2. 🌐 NGINX CVE-2026-42533 严重堆缓冲区溢出 — 跨 15 年版本范围，研究者声称可绕过 ASLR 实现 RCE | 评分：91

来源：The Hacker News / SecurityWeek / HKCERT / Beazley Security / Tenable 评分理由：CVSS v4 9.2、影响从 0.9.6（2011）到 1.31.2 的全版本范围、研究者挑战官方"仅 DoS"结论

F5 于 7 月 15 日修补 NGINX 三个内存安全漏洞，最严重的 CVE-2026-42533（CVSS v4 9.2 / v3 8.1）允许未认证远程攻击者通过精心构造的 HTTP 请求触发堆缓冲区溢出。漏洞存在于 NGINX 脚本引擎中，当 regex-based map 的输出变量与先前正则捕获引用（$1, $2）在同一字符串表达式中出现时，引擎的两遍评估机制在两次遍历之间共享捕获状态被覆写，导致测量遍分配的缓冲区过小。

F5 官方评估称 RCE 仅在 ASLR 禁用或可绕过时可能。但独立研究者 Stan Shaw（cyberstan）公开挑战此结论：在默认 Ubuntu 24.04 上，单次未认证 GET 即可通过反向捕获覆写泄露堆上未初始化地址数据，提供构建 RCE payload 所需的全部地址信息，声称测试中 10/10 成功。他同时指出 F5 的临时缓解（使用命名捕获替代匿名捕获）留有第二条溢出路径。修复版本：NGINX 1.30.4 / 1.31.3 / Plus 37.0.3.1。
